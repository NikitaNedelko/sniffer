# Краткий отчет по работе

## Что такое сниффер
Сниффер — это программа, которая перехватывает и анализирует сетевые пакеты, проходящие через сетевой интерфейс.

## Зачем он нужен
В учебных и диагностических целях сниффер помогает понять структуру сетевого трафика, проверить обмен данными и отладить сетевые приложения.

## Какие библиотеки используются
- `scapy` — перехват и разбор сетевых пакетов.

## Как работает программа
Скрипт `sniffer.py` запускает перехват пакетов через Scapy (`sniff`) и для каждого пакета:
- проверяет наличие IP-слоя;
- выводит IP-адрес отправителя, получателя и протокол;
- если есть TCP, выводит порты, флаги и номер последовательности;
- если есть UDP, выводит порты и длину пакета;
- если TCP/UDP нет, выводит, что пакет не TCP/UDP;
- обрабатывает неизвестные/нетипичные пакеты без аварийного завершения.

## Как запустить программу
1. Установить Python 3.x.
2. Установить Npcap или WinPcap (для Windows).
3. Создать и активировать окружение (через `uv`, если используется в проекте).
4. Установить зависимости:
   `pip install -r requirements.txt`
5. Запустить PowerShell от имени администратора и выполнить:
   `python sniffer.py --count 10`

Дополнительно можно указать интерфейс:
`python sniffer.py --count 10 --iface "Ethernet"`

## Пример вывода
[IP] 192.168.1.5 -> 8.8.8.8 | Protocol: UDP
[UDP] Source port: 5353 | Destination port: 53 | Length: 42

[IP] 192.168.1.5 -> 142.250.186.78 | Protocol: TCP
[TCP] Source port: 51432 | Destination port: 443 | Flags: PA | Seq: 123456

## Вывод по работе
Разработан простой консольный сниффер для Windows на Python и Scapy. Он перехватывает пакеты, выводит основные поля IP/TCP/UDP заголовков, поддерживает ограничение количества пакетов и устойчиво обрабатывает случаи без TCP/UDP.

Инструмент предназначен только для учебного использования в локальной сети или с разрешения владельца сети.
