# Отчет по разработке сниффера сетевых пакетов

## 1. Цель работы

Цель работы — разработать простой консольный сниффер сетевых пакетов на Python для Windows, который перехватывает сетевые пакеты в локальной сети и выводит основные поля IP/TCP/UDP-заголовков.

## 2. Постановка задачи

Необходимо реализовать программу, которая:
- запускается на Windows;
- использует библиотеку Scapy для перехвата сетевых пакетов;
- выводит информацию об IP/TCP/UDP-заголовках;
- поддерживает ограничение количества пакетов;
- позволяет фильтровать трафик по протоколу и порту;
- отображает результат в читаемом виде;
- сохраняет лог работы после завершения;
- сопровождается инструкцией по запуску и кратким отчетом.

## 3. Что такое сниффер

Сниффер — это программа для перехвата и анализа сетевого трафика, проходящего через сетевой интерфейс компьютера.

В учебных задачах сниффер позволяет:
- изучать структуру сетевых пакетов;
- наблюдать работу протоколов TCP/UDP/ICMP;
- проверять, какой трафик генерируют приложения;
- отлаживать сетевое взаимодействие.

## 4. Используемые технологии

- Python 3.x — основной язык разработки.
- Scapy — библиотека для перехвата и разбора сетевых пакетов.
- Rich — библиотека для красивого отображения таблицы в консоли.
- Npcap — драйвер/библиотека для захвата сетевого трафика на Windows.

Scapy на Windows требует Npcap или WinPcap, при этом Npcap является актуальным вариантом. Npcap предоставляет драйвер уровня ядра Windows для захвата и отправки пакетов. Rich используется для live-таблицы и автообновляемого терминального вывода.

## 5. Требования к запуску

Для запуска программы требуется:
- Windows 10/11;
- Python 3.x;
- установленный Npcap;
- установленные зависимости из `requirements.txt` (`scapy`, `rich`);
- запуск PowerShell от имени администратора.

Базовый запуск:

```powershell
.\.venv\Scripts\python.exe sniffer.py --count 100
```

## 6. Установка и проверка Npcap

Для работы сниффера на Windows требуется установленный Npcap, так как Scapy использует его для перехвата сетевых пакетов.

В рамках проверки Npcap уже был установлен на системе, поэтому повторная установка не выполнялась.

Проверка работы Npcap выполнялась командой:

```powershell
sc.exe query npcap
```

Если служба существует и находится в состоянии `RUNNING`, значит драйвер доступен для захвата сетевого трафика.

- `requirements.txt` — список зависимостей проекта.

## 8. Функциональность программы

Разработанная программа выполняет следующие функции:

- перехватывает сетевые пакеты с выбранного сетевого интерфейса;
- определяет тип пакета: TCP, UDP, ICMP или OTHER;
- выводит время получения пакета;
- отображает IP-адрес и порт отправителя;
- отображает IP-адрес и порт получателя;
- выводит TCP-флаги для TCP-пакетов;
- показывает размер пакета в байтах;
- отображает последние пакеты в live-таблице;
- считает статистику по TCP, UDP, ICMP и общему количеству пакетов;
- поддерживает фильтрацию по протоколу, порту и BPF-фильтру;
- сохраняет результат работы в txt или json.

## 9. Алгоритм работы программы

1. Пользователь запускает программу из PowerShell от имени администратора.
2. Программа считывает аргументы командной строки.
3. На основе параметров формируется BPF-фильтр.
4. Scapy запускает перехват сетевых пакетов.
5. Каждый пакет передается в обработчик.
6. Обработчик извлекает IP/TCP/UDP-поля.
7. Данные пакета сохраняются во внутреннее состояние программы.
8. Rich обновляет live-таблицу в консоли.
9. После достижения лимита `--count` или нажатия `Ctrl+C` перехват завершается.
10. Программа сохраняет лог и выводит итоговую статистику.

## 10. Аргументы командной строки

Программа поддерживает следующие параметры:

| Аргумент       | Назначение                                          |
| -------------- | --------------------------------------------------- |
| `--iface`      | выбор сетевого интерфейса                           |
| `--count`      | количество пакетов для перехвата, `0` — без лимита |
| `--filter`     | raw BPF-фильтр для Scapy                            |
| `--protocol`   | фильтр по протоколу: `tcp`, `udp`, `icmp`          |
| `--port`       | фильтр по порту                                     |
| `--max-rows`   | количество последних строк в live-таблице           |
| `--log-file`   | путь к файлу лога                                   |
| `--log-format` | формат лога: `txt` или `json`                       |

## 11. Примеры запуска

Базовый запуск:

```powershell
.\.venv\Scripts\python.exe sniffer.py --count 100
```

Фильтрация TCP/443:

```powershell
.\.venv\Scripts\python.exe sniffer.py --protocol tcp --port 443 --count 100
```

Фильтрация UDP/53:

```powershell
.\.venv\Scripts\python.exe sniffer.py --protocol udp --port 53 --count 100
```

Пользовательский BPF-фильтр:

```powershell
.\.venv\Scripts\python.exe sniffer.py --filter "host 8.8.8.8" --count 50
```

Сохранение лога в JSON:

```powershell
.\.venv\Scripts\python.exe sniffer.py --count 100 --log-format json
```
